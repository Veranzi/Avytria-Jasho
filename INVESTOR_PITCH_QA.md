#### Q: "If you're continuously listening for voice commands, what about user privacy? Are you recording everything?"

**A**: "Great question - privacy is our TOP priority. Here's how we handle it:

**1. Selective Implementation**:
- Continuous listening is **ONLY enabled for PWD users** who sign up with voice/face authentication
- Regular users get **tap-to-speak** (they tap mic button to activate)
- It's **opt-in during signup** - users explicitly consent
- Can be **toggled off anytime** in settings

**2. No Recording, No Storage**:
- We DON'T record conversations
- We DON'T store audio files
- Voice data is processed **on-device first** (local ML)
- Only command text sent to server (not audio)
- Audio deleted immediately after processing

**3. Wake Word Detection (Like Siri)**:
- App listens ONLY for wake word "Jasho"
- Everything else ignored until wake word detected
- Low-power mode (doesn't drain battery)
- Uses on-device ML (no data leaves device)

**4. Technical Implementation**:
```
User speaks ‚Üí On-device wake word detection ‚Üí 
If "Jasho" detected ‚Üí Start recording command ‚Üí
Send text (not audio) to server ‚Üí Process ‚Üí 
Delete all audio ‚Üí Return result
```

**5. Transparency**:
- **Privacy Policy** clearly explains this
- **Indicator dot** shows when listening (like iPhone)
- **Privacy dashboard** shows all voice commands (user can audit)
- **Delete data** anytime (GDPR compliant)

**6. Competitive Advantage**:
- **Siri, Alexa, Google Assistant** all do continuous listening
- We're MORE private (local processing, opt-in only, PWD-specific)
- Industry-standard approach, just applied to fintech

**Bottom Line**: PWD users CHOOSE continuous listening for convenience. It's like choosing Face ID over password - more convenient, equally secure."
